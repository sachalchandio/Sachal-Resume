The month I onboarded Astound, Starlink, Cox and friends end to end, built modular commission calculation into the sales pipeline, and rebuilt the chat experience.

## Onboarding carriers, top to bottom

A big chunk of November was adding new providers as first-class citizens of the sales system. Astound Broadband and Starlink both got full GraphQL CRUD, their own sale DTOs, migrations, and provider modules, and Cox picked up its own sale components and dialogs. Each carrier carries its own quirks, so I relaxed package validation where the data didn't fit a single rigid shape and threaded shipment fields through as optional where shipping is real.

- Astound and Starlink: full sale support, DTOs, and migrations
- Cox sale components, dialogs, and provider routing
- Shipping address and optional shipment fields on physical-good sales

## A commission engine that does the math

I built modular commission calculation into the sales module so payouts are computed consistently instead of by hand. That meant adding a CommissionPaid field across every sale DTO and its GraphQL surface, plus selectors in the sales-journey dialogs so the state is visible and editable. I also enriched the sale records with LOB package fields and PSU counts so the aggregation actually has the inputs it needs.

- Modular commission calculation methods on the server
- CommissionPaid threaded through every sale DTO and query
- Commission-paid selectors in the sales-journey UI

## Sharper search and indexing

Search got more precise this month. I added and backfilled a state column on the search index, improved provider mapping and the agent fields we index, and stopped pulling comments into the index so it stays lean and relevant.

- State column added and backfilled across the search index
- Better provider and agent fields in the index; comments excluded

## Chat, redesigned

On the frontend I reworked the chat list and tightened up the dashboards, including a 'New Sale' card on each provider dashboard and a unified treatment for sale-status legends and colors so the same status always looks the same everywhere.
