The month I built device inventory from create to assignment, made the dashboards update live over websockets, and taught the commission engine about payment delays and a commissionPaid flag.

## Inventory from scratch

I built the first version of the inventory subsystem so the team could track physical units instead of spreadsheets. That meant a create-inventory form, unit-list components, an assignment popup, available-stock counts, and filterable inventory requests.

- Create-inventory form and unit-list components
- Assign / available-stock tracking
- Filter input on inventory requests

## Dashboards that update themselves

Polling for fresh numbers was wasteful, so I moved the live data onto an event emitter and websocket subscriptions to push table updates as they happen. I also added system-monitoring panels and event-trend charts so the live stream had somewhere meaningful to land.

- Event emitter + websocket subscriptions for live tables
- System monitoring panels
- Event-trend charts and summary statistics

## Commissions that handle the real world

Payouts don't always fire immediately, so I added proper delay handling — including the edge cases where a zero-month delay should mean zero and a one-month delay was wrongly becoming two. I introduced a commissionPaid flag through the salary reports and detail views so paid commissions stop double-counting, and added a date-range filter to the dashboard summary.

- Fixed 0-month and off-by-one commission delays
- commissionPaid flag across salary reports and sale detail
- Active-agent filtering on salary processing
- Date-range support on the dashboard summary
